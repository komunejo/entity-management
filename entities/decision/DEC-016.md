title: the version numbers the skill, and only a minor carries new capability
status: accepted
date: 2026-07-15
tags: [governance, workflow]
---

`VERSION.md`, the engine's `ENGINE_VERSION` and the changelog entry all describe the skill. The `vX.Y.Z` tag marks the repository at the commit where that release was cut. They are not the same thing and they coincide only by convenience: a commit can add records, schemas or documentation to this repository without any of it reaching the version, because none of it reaches the package. The tag says *where*; the version says *what*.

Within the skill's version, a minor carries new capability — engine behaviour, schema language, the surface a user has to learn. A patch carries none: a defect fixed, a measurement redone, nothing that changes what the skill can do.

The practice has been consistent from the start and was simply never written: 0.2.0 brought aggregate constraints and the contract policy, 0.2.1 brought only a re-measured SKILL.md description; 0.4.0 brought three capabilities, 0.4.1 brings one fix. 0.3.0 is the case that tests the rule rather than illustrating it — twelve reproduced defects fixed, which is patch material, arriving together with stricter meta-validation and three workflows the documentation had promised and never taught. It is the strictness and the workflows that earn the minor there, not the twelve fixes, and the rule survives the one release that could have broken it. Until now the rule could only be recovered by reading the shape of the changelog and inferring backwards — which is exactly the reconstruction this project exists to make unnecessary. An unwritten rule is one the next release guesses at, and a version number nobody can rely on is worse than none, because it still looks like a promise.

What `1.0` would mean is deliberately not decided here, and the pre-1.0 licence to break things is not claimed either. No release so far has broken anything, so there is no practice to record — and inventing one would repeat the error this decision closes.
